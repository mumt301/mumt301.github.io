JavaScript Intro

When you access a website, the website is very likely made out of three components

  • HTML code, used to indicate the content of the website.
  • CSS code, used to indicate the aesthetics (and layout) of website.
  • JavaScript code, used to control (or alter) the behavior of the website.

JavaScript is the programming language of the web All modern web browsers support it

JavaScript can change the content and attributes of HTML elements:

1. Running JavaScript code Time to fire up your JavaScript interpreter

  • You can (luckily) access a JavaScript interpreter from your favorite web browser.
  • The tools offered by modern browsers allow you to debug (finding and correcting errors) the JavaScript code of your website.
  • You can achieve this by using the JavaScript console (interpreter), reading the HTML source, and CSS attributes.
  • One common way of debugging is using the console.log() from your JavaScript code to display values and messages in the console.

2. JavaScript values Handling different data types

Most languages have a set of supported data types. JavaScript supports the following ones:

  • Number
  • String
  • Boolean
  • null & undefined
  • Array
  • Function
  • Object

Number values

Let's start with numbers.

Remember, in the end, computers are simply blazing fast calculators.

2 + 2;

Some languages distinguish between integer and real numbers (real numbers are often called floats, due to the floating point technique used to represent them).

JavaScript does not make that distinction, everything is a real number (or simply, a Number).

When explicitly using decimal values, you may experience issues with precision.

Try to work with integers as much as you can, especially when you compare quantities in conditions and loops. 

0.1 + 0.2;

if (0.1 + 0.2 === 0.3) {
  console.log("Yes!");
}
else {
  console.log("Oh no, precision issues!");
}

if (1 + 2 === 3) {
  console.log("Yes!");
}
else {
  console.log("Oh no, precision issues!");
}

String values

Strings are simply, a text value.

Calling text values "strings" may be confusing, but it is a well-established term in Computer Science that appears in almost every programming language nowadays.

Therefore, we will be using the term string a lot.

We have three ways for describing strings in JavaScript: 

`Using backticks`;
"Using doublequotes";
'Or using singlequotes';

All of them describe a string value.

They have slight differences but nothing to worry about for now.

Use any convention you want, but the backticks (`these ones`) will let us format our strings later, so maybe get used to these ones.

Escaping characters

Some characters are difficult to write in a string, for one reason or another.

These characters are often escaped with a backslash \.

A tab is encoded as \t. A newline is encoded as \n. In some instances (e.g., in the terminal of your operating system), a space is escaped as \ (backslash followed by a space).

You will almost never need to escape a space in JavaScript, but you will need to escape tabs and newlines.

Boolean values

1 or 0. Yes or no. Used a lot to control the flow of programs: 

3 > 2;

if (301 > 201)

    console.log("MUMT 201 is a requirement for MUMT 301");

null and undefined

null indicates that something deliberately does not have a value.

undefined indicates that something is uninitialized or undefined.

Don't worry about these for now. You will see later in which cases they show up.

Array values

An array is a collection of values.

We denote an array by writing comma-separated values within two square brackets.

["Rhythm guitar", "Lead guitar", "Bass guitar", "Drums", "Vocals"];

Arrays have certain attributes or properties. For example, the length, which tells us how many values are stored in the array.

["Rhythm guitar", "Lead guitar", "Bass guitar", "Drums", "Vocals"]['length'];

We access the attributes of a value using square brackets after the value and referring to the attribute's name.

If the attribute name meets the rules of valid identifiers, they can also be accessed using a period after the value, followed by the name of the attribute. 

["Rhythm guitar", "Lead guitar", "Bass guitar", "Drums", "Vocals"].length;

Other datatypes also have attributes. They are not exclusive of arrays.

For example, Strings have a startsWith attribute to verify whether they start with a given substring.

FunctionsA box that receives an input and produces an output

function addition (a, b) {
  return a + b;
}

addition(2, 2);

You can think of functions in several ways:

  • Functions are useful to abstract behaviors.
  • They are also useful to recycle code that you plan to run in several different places.
  • They allow you to extend the "vocabulary" of the JavaScript language, so that it is capable of doing more complicated things.

In practice, JavaScript functions are more complex than this, however, we can assume this is how they operate for now.

Objects

Objects store information in key: value pairs of values.

They are very useful to organize information in intuitive ways.

let guitar = { 
  model: "Soloist", 
  brand: "Jackson", 
  year: 2016, 
  frets: 24 
};

Each of the fields can be accessed as an attribute:

console.log(guitar['brand']);

And new attributes can be added too:

guitar.pickups = "Dimarzio Black Winter";

Notice that the fields could be of different data types:

typeof(guitar.pickups);
typeof(guitar.frets);

3. JavaScript variables (Bindings) Variables are identifiers that we can use to store data values

We store values in variables.

There are three words that we use to declare variables: 

let x = 5;
const y = 5;
var z = 5;

We will mostly use let.

const is used for assigning values that you don't expect will change.

var is there for backward compatibility with older JavaScript versions. It's almost identical to let. 

let mouse = "mickey";

console.log(mouse);

We can reassign the variable to a different value (except for constants declared using const).

mouse = 1;

console.log(mouse);

Notice that

console.log(mouse);

is different than

console.log("mouse");

This is because the variable mouse is not a value, it is a named reference to a value.

What if we try this one:

console.log(duck);

It doesn't exist. But we can make it exist

let duck = "donald";
console.log(duck);

We already used a variable for storing the guitar object.

We can really store any data type in a variable:

  • Numbers
  • Strings
  • Booleans
  • Arrays
  • Objects
  • even the Functions

4. Control structures Controlling the paths of your code

Conditions

When we run computer code, we can decide to go through one branch of code or another one.

In order to control the flow, we use the conditional statements if and else: 

function noteSpelling(inputNote) {
  if (inputNote === 'F#') {
    console.log('This is an F sharp.');
  }
  else if (inputNote === 'Gb') {
    console.log('This is a G flat.');
  }
  else {
    console.log('Sorry, I only know F sharps and G flats.');
  }
}
      
let note = 'F#';
noteSpelling(note);

note = 'Gb';
noteSpelling(note);

note = 'C#';
noteSpelling(note);

Loops

Beside conditions, another common operation to control the flow of the program is doing loops.

There are three ways of achieving this in JavaScript:

  • for loops
  • while loops
  • do while loops

In general terms, you can achieve exactly the same operation using either of them.

Some of them are simply more convenient in certain scenarios.

Here is a while loop: 

const ageLimit = 18;
      
let myAge = 0;

while (myAge < ageLimit) {
  console.log(`You are ${myAge} years old. You can't drink.`);
  myAge = myAge + 1;
}

console.log(`You are ${myAge} years old. You can drink now. Welcome.`);

This is the equivalent code using a for loop: 

const ageLimit = 18;

for (var myAge = 0; myAge < ageLimit; myAge++) {
  console.log(`You are ${myAge} years old. You can't drink.`);
}

console.log(`You are ${myAge} years old. You can drink now. Welcome.`);

You can read more about for loops here.

6. The script tag Embedding code inside an html file

Remember the style tag and how we used it to add CSS to our HTML website?

The equivalent for javascript is the script tag.

  • Lines within the script tag contain JavaScript code, and the browser will interpret the code between them as such.
  • Scripts can be placed in any section of HTML code, however putting scripts at the bottom of the body section of a web page can reduce display time.
  • Separating HTML and JavaScript by putting all the code in one place, is always a good habit.
  • Similarly to CSS files, JavaScript can also be placed in external files. This is practical when reusing the same code.
  • JavaScript files have the file extension .js. To use an external script, put the name of the script file in the source (src) attribute of the script tag:
<script src="myScript.js"></script>

7. The Document Object Model Your HTML code as a tree structure

The Document Object Model (DOM) is a platform and language-neutral interface that allows programs and scripts to dynamically access and update the content, structure, and style of a document.

  • When a web page is loaded, the browser creates a Document Object Model of the page.
  • The DOM is constructed as a tree:

You can access your HTML document within JavaScript through the document object.

You can think of document as a variable that is already given to you, and it provides access to your HTML elements.

With it, you can select elements from the DOM, change their attributes, and append new elements to it. 

function domExample() {
  let aButton = document.getElementById("domButton");

  aButton.classList.add("primary");
  aButton.innerHTML = "I clicked it!";      
}

For example, when the button below is linked to the domExample() function. It will run its JavaScript code.

The button is defined in the following way 

<button id="domButton" class="button" onclick="domExample()">Example</button>

Let's apply the concepts we learned to create a table at the end of this document.

We will use an Array of Objects to store (a portion of) the discography of The Beatles.

We will iterate through them using a for loop to generate the rows of a table element in HTML.

The table element does not exist yet!

We will attach it to a container element with an id="discography_table".

The container element is the last html element of this file, before the script tag with all the JavaScript code that we have been using so far.

You cannot see the container element, however, after you press the button, our loop will populate it with our new table.

Here is the code responsible for creating and filling up the table. 

function discography() {
  let table = document.createElement('table');
  table.id = "beatles_table"
  for (row = 0; row < beatles_releases.length; row++) {
    tr = document.createElement('tr');
    let release_data = Object.keys(beatles_releases[row]);
    for (col = 0; col < Object.keys(beatles_releases[row]).length; col++) {
      td = document.createElement('td');
      tr.appendChild(td);
      td.innerHTML = beatles_releases[row][release_data[col]];
    }
    table.appendChild(tr);
  }
  document.getElementById("discography_table").appendChild(table);
}
