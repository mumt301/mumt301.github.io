Forms and inputsBasic elements for defining a form

HTML forms are used to collect user input. HTML forms can contain input elements like text fields, checkboxes, radio-buttons, submit buttons, and more.

Forms' syntax uses the form tag to create an HTML form. Within the form, the input element is used to select user information. The input element has the following attributes:

  • The type attribute is used to specify the type of data (text field, radio button, checkbox, submit button, etc.). HTML Input Types link in W3 Schools
  • The name attribute stores the user-entered data.
  • The id attribute, useful when using the label element (as will be explained later).

HTML Forms and Input link in W3 Schools

A simple example of an input text field would be:

<form method="get">
  <p>User name:
    <input type="text" name="userName">
  <br>
  <input type="submit">
</form>

A form can contain different types of input elements (e.g., text, password fields, radio-buttons, checkboxes, ...)

Example of radio buttons

    <input type="radio" name="gender" value="other">Other

</form>

The label element

The label element is useful to provide a label for the input elements of the form. Regarding its attributes:

  • To connect the input element to its label, the for attribute of the label element should be equal to the id attribute of the input element.

Label elements are useful for screen-reader users and users that have trouble clicking on very small regions (e.g., radio buttons and checkboxes). The user can toggle the radio button or checkbox by clicking on the text corresponding to the label.

Here it is an example including everything seen before, plus the label element notice how easy it is to select any option?

<form method="get">
    <label for="text_username">Username:</label>
    <input type="text" name="username" id="text_username">

    <input type="radio" name="gender" value="male" id="radio_male">
    <label for="radio_male">Male</label>

    <input type="radio" name="gender" value="female" id="radio_female">
    <label for="radio_female">Female</label>

    <input type="radio" name="gender" value="other" id="radio_other">
    <label for="radio_other">Other</label>

    <br>
    <label for="text_password">Password:</label>
    <input type="password" name="psswd" id="text_password">

    <br>
    <label for="text_artist">Artist name:</label>
    <input type="text" name="artist" id="text_artist">

The HTTP GET method and using the data from forms in JavaScript

The method attribute of the form element specifies the HTTP method used when submitting the form data. When method="get", the form data is sent as URL variables. GET is the default HTTP method when submitting form data. "The Method Attribute" section from HTML Forms Attributes link at W3 Schools

We usually want to use the data a user entered into a form. In order to accomplish this, we parse the values of the parameters in the URL.

Create a form where the user can enter the name of a city. For now, when submitting the form, the only thing returned to the user will be their own input (the name of the city entered). In Exercise # 2, we will return the current weather of the user-entered city using an API. Let's focus for now on the first part of this exercise.

Create a form for asking the name of a city for which the user is interested.

When the user presses the submit button, the name of the city should be printed on the screen.

Steps

  1. Create the webpage skeleton
  2. Create the form
    Include a submit button. After clicking it, the values sent through the form should be shown as parameters in the URL.
  3. Create an HTML element that will act as a placeholder for rendering the results
    You will need to assign it an id to identify this element later and populate it with the resulting data.
  4. Create a function that gets called when submitting data through the form
    We will use the window.onload event (seen in class 05) to call the function.
  5. The function should read the data submitted in the form and store the city name in a variable
    We will parse the parameters contained in the URL using the (new URL(document.location)).searchParams property and its get method (also seen in class 05).
  6. Print the user-entered city in the console
  7. From within the function, populate the placeholder element (to display the city name in the browser)
Remember to use the JavaScript console to check your variables and debug your code!
